# 前端设计与架构文档

## 1. 项目概述

-------

本项目是一个基于大语言模型（Agent）驱动的智能旅行规划平台的前端模块。用户通过结构化表单输入出行约束（时间、预算、目的地等），前端通过调用底层的 Agent API，生成并展示包含航班、酒店、景点的完整时间轴行程。

v1.0 核心目标：跑通核心生成链路，确立独特的视觉规范，实现桌面端的高效交互与行程的 PDF 导出。

## 2.技术栈与环境配置

-----------

* **核心框架：** Vue 3 (Composition API, `<script setup>`)

* **状态管理：** Pinia（用于管理深层嵌套的行程数据和全局 AI 加载状态）

* **路由管理：** Vue Router 4（处理主页面与独立地图页的跳转）

* **CSS 方案：** Tailwind CSS + shadcn-vue（极致的样式定制自由度，完美契合 Noir-grunge 风格）

* **辅助工具：** `html2pdf.js` 或 `jspdf` (用于导出 PDF)

## 3.视觉与 UI 规范 (UI/UX Guidelines)

-------------------------------

本项目在视觉上采用 **Surreal Noir-Grunge** 风格，打破传统旅游应用的鲜艳明快，营造一种深邃、克制、充满探索感的氛围。

* **色彩调色板 (Color Palette)：**
  
  * 主背景：Void Black (深渊黑，如 `#0a0a0a`)
  
  * 主文本/线框：Chalk White (粉笔白，如 `#f4f4f5`)
  
  * 辅助灰阶：使用一系列冷灰色调（用于时间轴、禁用状态、次要文本）

* **排版 (Typography)：**
  
  * 标题/重点数据：Distressed Serif (做旧风格的衬线体，如 _Playfair Display_ 或 _Cinzel_，可结合 CSS `text-shadow` 制造斑驳感)
  
  * 正文/信息流：干净利落的无衬线体 (如 _Inter_ 或 _Geist_)

* **纹理与特效 (Textures & Effects)：**
  
  * **噪点遮罩 (Film Grain)：** 在全局应用一个 `pointer-events: none` 的 SVG 噪点背景图层，增加胶片质感。
  
  * **星际线框 (Celestial Wireframes)：** 弱化传统的实心卡片背景，大量使用极细的白色边框（1px solid border）和几何线条来分割内容区域。
4. 核心页面与组件拆解 (仅桌面端)

-------------------

## 4.1 主工作区 (`ItineraryBuilder.vue`)

采用经典的左右分栏布局（类似于专业生产力工具）。

**左侧：表单控制台 (Control Panel)**

* **目的地标签组 (`DestinationTags`)：** 扁平化的黑白胶囊样式，支持回车添加、点击删除。

* **数字/范围输入 (`ConstraintInputs`)：** 预算和出行人数使用极简的下划线输入框或刻度滑块。

* **触发按钮 (`GenerateButton`)：** 高反差的黑底白字按钮，触发时进入 `disabled` 状态。

* **AI 状态栏 (`AiStatusBar`)：** 位于左下角固定位置。这是一个关键组件。当 Agent 开始工作时，这里会像命令行终端一样滚动显示当前进度（例如：`> System: Fetching flight data for BKK...` -> `> Agent: Analyzing hotel options...`）。这种设计完美契合黑客/赛博氛围。

**右侧：数据展示区 (Display Canvas)**

* **空状态/加载态 (`LoadingSkeleton`)：** 未生成时，展示星际线框风格的骨架屏。

* **全局概览 (`SummaryBar`)：** 顶部展示总预算、天数等宏观数据。

* **工具栏 (`ActionToolbar`)：** 位于右侧顶部或底部，包含一个显眼的 **[ 导出为 PDF ]** 按钮。

* **景点栏（Attraction Tags）双态逻辑设计**
  
  这部分是用户输入的核心。前端需要维护一个景点数组 `attractions: string[]`。
  
  * **UI 表现：** 在目的地（如 Bangkok, Phuket）下方，提供一个输入框。用户输入回车生成 Tag（黑底白框，带有 `x` 删除键）。
  
  * **请求 Payload 改造：**JSON
    
        {
          "destinations": ["Bangkok", "Phuket"],
          "must_visit_attractions": ["The Grand Palace", "Patong Beach"], // 用户填写的Tag
          // ... 其他参数
        }
  
  * **双态 Prompt 调度逻辑（前端概念映射）：**
    
    * **主动态 (数组长度 > 0)：** Agent 收到强约束，必须将这几个景点作为“锚点”排入行程，并围绕它们补充合理的餐饮和交通。
    
    * **探索态 (数组长度 === 0)：** 前端可以显示一行微弱的提示语：“_Leave blank for AI serendipity..._”（留白以获取 AI 带来的惊喜）。此时 Agent 将根据“时间+预算”自由发挥，自动生成包含景点 Tag 的完整行程。
- **展示面板：水平时间轴 (Horizontal Timeline) 设计** 
  采用**水平滚动 (Horizontal Scroll)** 的宽画幅展示。**整个容器是一个横向可滑动的视窗：**
  
  1. **全局中轴线 (The Axis)：**
  * 一条绝对居中（`top: 50%`）的水平虚线（`border-dashed`），颜色为带透明度的白色（如 `rgba(255,255,255,0.3)`），贯穿整个屏幕。
  2. **一级时间节点 (Primary Date Markers)：**
  * 作为每一天的起始标兵。比如大写的 **MAR 26**。它直接坐在虚线上，或者将虚线截断，采用高对比度的字体（如粗体无衬线）。
  3. **活动节点 (Activity Nodes)：**
  * 串联在虚线上的“珠子”。节点类型分为：✈️ 航班、🏨 酒店、🖼️ 景点。
  
  * **视觉对齐：** 所有的 Emoji 图标或景点缩略图的**垂直中心点**，必须精准压在水平虚线上。
  4. **二级时间 (Secondary Time)：**
  * 精准定位在每个活动节点图标的**左上方**（例如 “19:00”），使用极小的字号（如 10px 或 12px）和较暗的灰色，营造精致的数据感。
  
  ### 2.2交互细节考量 (UX Details)

- **数据映射：** 后端传回来的 `flights`, `hotels`, `views` 是三个独立的数组。前端需要写一个**数据清洗函数**，将它们合并成一个单一数组，并按照 `datetime` 统一排序，最后再按 `date` (天) 进行分组，才能顺利渲染出上述的 DOM 结构。

- **横向滚动的引导:** 需在容器左右两侧添加悬浮的、半透明的左右箭头 `<` `>` 按钮，或者支持鼠标滚轮（Mouse Wheel）映射为横向滚动。

- **信息的渐进式披露：** 面板上只展示最核心的时间、名称和缩略图。当用户的鼠标 `hover` 或者 `click` 某个节点（如航班）时，可以从节点上方或下方弹出一个极简风格的 Popover（气泡卡片），展示行李额度、航站楼、预算等详细数据。

### 4.2 独立地图视窗 (`MapView.vue`)

* 当用户点击某个地点的 "查看地图" 按钮时，通过 Vue Router 跳转到此页面（禁止新开标签页、要求单独加载一个悬于主页面上方的小页面）。

* 全屏展示交互式的 Google Maps iframe，配合极简的返回按钮和深色模式（Dark Mode）地图样式，保持整体 UI 调性一致。

## 6. v2.0 规划池 (Future Roadmap)

----------------------------

为了保持 v1.0 的专注，以下高优功能已明确移入下一个开发周期：

1. **移动端适配 (Responsive Design)：** 引入抽屉 (Drawer) 组件，重构双栏布局。

2. **局部重绘与微调 (Local Redraw)：** 允许针对时间轴上单一节点（如更换某一家酒店）进行二次 Agent 调度。

3. **自然语言个性化约束 (NLP Input)：** 增加 Textarea，允许用户输入非结构化需求（如“想吃夜市”），进一步发挥大模型优势。
